The same disciplined loop every time — whether it's a single phishing mail or a multi-host intrusion.

Come back stronger

Incident Response

Endpoint intrusion contained through CrowdStrike–QRadar–firewall correlation

Problem

A CrowdStrike Falcon endpoint detection needed validation — endpoint signal alone lacked the network and identity context to judge whether it was a real intrusion or noise.

Approach

Correlated Falcon detections with IBM QRadar logs and FortiGate firewall traffic, tracked the incident P1–P3 in ServiceNow, mapped the activity to MITRE ATT&CK, contained the host via Falcon, and ran root-cause analysis with preventive recommendations.

Outcome

Threat contained and root-caused; detection feedback and tuning reduced repeat false positives across the SOC.
